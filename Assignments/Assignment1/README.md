# Assignment 1 – Understanding Development Environments and Tools for Edge AI and IoT

## Problem
The purpose of this assignment was to understand the different development tools and environments used in Edge AI and IoT systems. Since edge devices have limited hardware and resources, the challenge is selecting tools that can efficiently support coding, data processing, machine learning, and deployment without relying heavily on cloud computing.

## Approach
To address this, I researched and analyzed several commonly used tools in Edge AI and IoT development. These included Visual Studio Code, Node.js, Edge Impulse CLI, TensorFlow and TensorFlow Lite, Google Colab, and generative AI coding tools like GitHub Copilot.

For each tool, I examined its purpose, key features, and how it is used in real-world applications. I also explored practical examples, such as how these tools support healthcare systems, wearable devices, and smart IoT environments. This approach helped connect theoretical concepts to real-world use cases.

## Results
The analysis showed that each tool plays a specific role in the development process. For example, Visual Studio Code supports coding and debugging, Node.js enables efficient handling of data from multiple devices, and TensorFlow Lite allows machine learning models to run directly on edge devices.

The combination of these tools allows developers to build, test, and deploy AI systems that operate efficiently on limited hardware. This assignment provided a foundational understanding of the tools used in Edge AI and IoT systems. It showed how different technologies work together to support the full development process, from coding and testing to deployment on edge devices.

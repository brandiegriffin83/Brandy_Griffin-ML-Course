# Lab 3 – Deploying an AI Model in a Simulated Edge Environment

## What I Did
Instead of working in a notebook environment, I used Visual Studio Code to build and deploy a machine learning model. I trained a convolutional neural network using the MNIST dataset, converted the trained model into TensorFlow Lite format, and then tested it within a simulated edge environment using Edge Impulse.

## What I Learned
Through this experience, I gained a deeper understanding of how AI systems move from development to deployment. I learned how to manage dependencies across different tools, handle environment setup in Visual Studio Code, and evaluate performance factors like latency and efficiency. This lab also showed me how important compatibility is when working with multiple technologies.

## Challenges
One of the most difficult parts of this lab was troubleshooting installation and configuration issues. I encountered several errors related to Node.js versions and missing modules while setting up Edge Impulse CLI. Resolving these problems required uninstalling and reinstalling tools, clearing caches, and carefully following setup instructions. Staying patient and working through each issue step by step was a key part of completing the lab.

## Results
After resolving the setup issues, the model successfully trained with over 98% accuracy and was converted into TensorFlow Lite format. I was able to deploy and test the model in a simulated edge environment, where it performed efficiently with minimal delay.
